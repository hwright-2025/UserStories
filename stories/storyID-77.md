Each user story is an informal, general explanation of a desired ecosystem feature or process written from the perspective of a specific end user. The purpose of a User Story is to articulate how a piece of work will deliver a particular value back to the user. User stories are gathered from the NICHD community and are used to initiate Ecosystem efforts.

# S77: As an NICHD-funded researcher, I want access to a tool informed by repository assessments so that I can select a repository that aligns with NIH DMS Policy and supports my data type.

<br>

| User Story | Current Problem | User Goal
| :------------- | :------------ | :------------ |
| What does the user want to be able to do? | Why can't the user do this today? | What's the user's ultimate goal? 
|As an NICHD-funded researcher, I want access to a tool informed by repository assessments so that I can select a repository that aligns with NIH DMS Policy and supports my data type. | Today, researchers are often unaware repositories most suitable for their data type   | My goal is to list details relevant to the DMS Policy in the Data Repository Finder to help researchers write their DMS Plans. |
